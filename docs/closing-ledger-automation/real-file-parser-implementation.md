# 브랜아크 발주서 실파일 파서 구현 메모

## 현재 문제

- 기존 구현은 발주서 파일을 Drive에 저장한 뒤 화면 입력값으로 테스트 행 1건만 작성했습니다.
- 상품명, 수량, 공급단가를 수동 입력해야 해서 실제 발주서 자동화 목적과 맞지 않았습니다.
- 단가표는 접근 확인 수준이었고, 실상품명 기준 매칭과 반영 차단 로직이 없었습니다.

## 변경된 처리 흐름

```text
발주서 파일 업로드
-> Drive 저장
-> csv / xlsx / xls 분석
-> 후보 시트 탐색
-> 컬럼 정규화
-> 상품명 정규화 및 수량 해석
-> 상품별 집계
-> 단가표 매칭
-> 일일마감 반영 예정 행 구성
-> 검증 통과 시 일일마감 시트 반영
```

## 파일 파싱 방식

- `csv`
  - Apps Script에서 바로 `Utilities.parseCsv`로 읽습니다.
- `xlsx`, `xls`
  - 업로드 blob을 Drive API 고급 서비스로 임시 Google Sheet로 변환한 뒤 시트 데이터를 읽습니다.
  - 임시 변환 파일 ID는 응답에 포함할 수 있고, 기본값은 처리 후 휴지통으로 이동합니다.

## 후보 시트 탐색 규칙

- 우선 시트명 후보
  - `운송장`
  - `출고일지`
  - `출고일지(2)`
- 동시에 상단 1~10행에서 아래 핵심 헤더를 탐색합니다.
  - 상품명
  - 수량
  - 주문번호
  - 판매처
  - 옵션명
- 빈 시트는 제외합니다.

## 단가표 매칭 방식

- `PRICE_SHEET_ID`가 Google Sheet면 첫 번째 시트에서 직접 읽습니다.
- xlsx/xls 파일이면 임시 Google Sheet 변환 후 읽습니다.
- 매칭 우선순위
  1. 원본 상품명 완전 일치
  2. 정규화 상품명 일치
- 단가를 찾지 못하면 일일마감 반영을 차단합니다.

## 일일마감 반영 방식

- `DAILY_SHEET_ID` 첫 번째 시트를 사용합니다.
- 헤더 행을 상단 1~10행에서 탐색하고, 아래 컬럼을 이름 기준으로 찾아 입력합니다.
  - 주문일
  - 상품명
  - 수량
  - 공급단가
  - 공급가 합계
  - 비고
- 공급가 합계는 `수량 * 공급단가`로 계산합니다.
- 비고에는 원본 파일명, 원본 시트명, 처리 시각, Drive file ID를 남깁니다.

## 검증 시나리오

- 상품명 헤더 또는 값이 없으면 차단
- 수량 헤더가 없거나 숫자가 아니면 차단
- 단가표 매칭 실패 시 차단
- 일일마감 시트 접근 실패 또는 헤더 누락 시 차단
- 파일 변환 실패 시 차단
- 최종 반영 행이 0건이면 차단

## Apps Script 재배포

- 이 구현은 Apps Script 코드 변경이 포함되므로 새 버전으로 재배포가 필요합니다.
- xlsx/xls 변환을 위해 Drive API 고급 서비스 활성화 여부도 함께 확인해야 합니다.
