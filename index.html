GitHub Actions의 Branark Closing Ledger API Env Check workflow를 수동 실행해 Drive 폴더, 일일 마감 양식, 단가표 접근 가능 여부를 확인합니다.

테스트 워크플로우

파일 업로드 후 자동화가 처리해야 하는 순서입니다. 피드백 시 이 흐름이 실무와 맞는지 확인하면 됩니다.

  1. 1. 발주서 파일 첨부xlsx / xls / csv
  2. 2. 파일 형식 검사확장자 확인
  3. 3. 중복 파일명 확인Drive 폴더 기준
  4. 4. Google Drive 저장DRIVE_FOLDER_ID
  5. 5. 발주서 내용 확인운송장 / 출고일지
  6. 6. 단가표 매칭PRICE_SHEET_ID
  7. 7. 일일 마감 양식 작성DAILY_SHEET_ID
  8. 8. 작업 결과 화면 표시성공 / 오류 / 단가표
2

발주서 업로드 테스트

파일을 선택하면 화면에서 파일명을 확인하고, 업로드 및 검증 시작 버튼으로 테스트 흐름을 확인합니다.

현재는 테스트 UI 단계입니다. 실제 Drive 저장과 Sheet 작성은 API 구현 및 workflow 연결 후 동작합니다.
