# 마감원장 검증 규칙 초안

## 1. 상품명 정규화 규칙

발주서의 상품명은 옵션 수량을 포함하는 경우가 많으므로, 단가 매칭 전에 정규화합니다.

### 기본 규칙

| 입력 예시 | 정규화 상품명 | 수량 |
|---|---|---:|
| `아삭아삭 궁채장아찌 1kg_10` | 아삭아삭 궁채장아찌 1kg | 10 |
| `아삭아삭 궁채장아찌 4kg_4` | 아삭아삭 궁채장아찌 4kg | 4 |
| `김밥집 궁채 2kg_3` | 김밥집 궁채 2kg | 3 |
| `전통가마솥간장게장 650g_2+ 양념게장 300g_2` | 전통가마솥간장게장 650g / 양념게장 300g | 각각 2 |

### 예외 처리

- 상품명 끝에 `_숫자`가 있으면 해당 숫자를 실제 개수로 봅니다.
- `_숫자`가 없으면 기본 수량은 1개로 봅니다.
- `+`로 묶인 세트 상품은 구성 상품을 분리하여 각각 수량을 계산합니다.
- 띄어쓰기 오류는 가능한 범위에서 정규화합니다.
  - 예: `아삭 명이나물` / `아삭명이 명이나물` → 같은 상품으로 매칭 후보 표시

---

## 2. 단가 매칭 규칙

단가표는 정규화된 상품명을 기준으로 매칭합니다.

### 매칭 우선순위

1. 완전 일치
2. 공백 제거 후 일치
3. 괄호 문구 제거 후 일치
4. 사전 등록된 별칭 매칭
5. 유사도 매칭 후보 표시

### 주의사항

- 유사도 매칭은 자동 확정하지 않습니다.
- 유사도 매칭 상품은 PR 검증 리포트에 `확인 필요`로 표시합니다.
- 단가가 없는 상품은 금액 계산에서 제외하고 누락 목록에 표시합니다.

---

## 3. 수량 검증 규칙

아래 항목은 자동 검증 리포트에 표시합니다.

- 상품별 원본 행 수
- 옵션 반영 후 실제 수량
- 동일 상품 합산 수량
- 세트 상품 분리 수량
- 총 수량

### 오류 의심 기준

| 항목 | 처리 |
|---|---|
| 수량이 0 이하 | 오류 |
| `_` 뒤 문자가 숫자가 아님 | 확인 필요 |
| 같은 상품인데 표기명이 2개 이상 | 확인 필요 |
| 단가표에 없는 상품 | 오류 또는 확인 필요 |
| 세트 상품인데 구성 분리가 안 됨 | 확인 필요 |

---

## 4. PR merge 전 확인 체크리스트

- [ ] 발주서 원본 파일이 맞는지 확인
- [ ] 상품별 합산 수량 확인
- [ ] 옵션 수량 계산 확인
- [ ] 세트 상품 분리 계산 확인
- [ ] 단가 누락 상품 확인
- [ ] 거래처별 합계 확인
- [ ] 총 금액 확인
- [ ] 검수자 코멘트 작성

---

## 5. merge 제한 권장 기준

아래 항목이 있으면 원칙적으로 merge하지 않습니다.

- 단가 누락 상품 존재
- 수량 파싱 실패 상품 존재
- 세트 구성 분리 실패
- 발주서와 초안 총 행 수 불일치

단, 긴급 처리 시 PR 코멘트에 예외 사유를 남긴 뒤 merge합니다.
