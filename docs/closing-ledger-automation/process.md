# 마감원장 자동화 업무 흐름

## 1단계. 발주서 입력

운영자는 아래 두 방식 중 하나로 발주서를 입력합니다.

### A안. GitHub 업로드

- `orders/incoming/` 폴더에 발주서 파일 업로드
- 파일명 예시: `20260702_식봄_발주서.xlsx`
- GitHub Action이 업로드를 감지하여 처리

### B안. Google Spreadsheet 감지

- 지정된 Google Drive 또는 Spreadsheet를 감지
- 신규 발주서가 확인되면 GitHub Action 또는 별도 스케줄러가 처리

> 초기 구축은 A안 GitHub 업로드 방식으로 시작하고, 안정화 이후 B안을 추가하는 것을 권장합니다.

---

## 2단계. 데이터 비교 및 단가 매칭

자동화 스크립트는 발주서와 발주단가표를 읽고 아래 작업을 수행합니다.

1. 상품명 정규화
   - 예: `아삭아삭 궁채장아찌 1kg_10` → 상품명 `아삭아삭 궁채장아찌 1kg`, 수량 `10`
   - 예: `김밥집 궁채 2kg_4` → 상품명 `김밥집 궁채 2kg`, 수량 `4`

2. 동일 상품 합산
   - 같은 상품은 수량을 합산하여 한 줄로 정리

3. 발주단가표 매칭
   - 정규화된 상품명을 기준으로 단가표에서 단가 조회

4. 금액 계산
   - `수량 × 단가 = 공급가 또는 발주금액`

5. 오류 표시
   - 단가 누락
   - 상품명 매칭 실패
   - 수량 파싱 실패
   - 중복 의심 상품

---

## 3단계. PR 검증

자동화 결과는 바로 마감원장으로 확정하지 않고 PR로 생성합니다.

PR에는 아래 산출물이 포함됩니다.

- 마감원장 초안 CSV
- 상품별 합산 결과
- 단가 매칭 결과
- 누락 상품 목록
- 검증 리포트 Markdown

담당자는 PR에서 아래를 확인합니다.

- 상품 수량이 발주서와 일치하는지
- 옵션 수량 계산이 맞는지
- 단가가 올바르게 매칭되었는지
- 단가 누락 상품이 없는지
- 거래처별 합계가 맞는지

---

## 4단계. merge 후 최종 시트 생성

PR 검증이 완료되어 main 브랜치에 merge되면 최종 작업이 실행됩니다.

1. Google Spreadsheet 마감원장 생성
2. 거래처별 시트 생성
3. 전체 합계 시트 생성
4. 누락 또는 경고 항목 별도 표시
5. 텔레그램으로 완료 알림 발송

---

## 운영 원칙

- 돈과 정산이 연결된 작업이므로 자동 생성 즉시 확정하지 않습니다.
- PR은 반드시 사람 검수 단계로 사용합니다.
- 단가 누락이 있으면 원칙적으로 merge 전 보완합니다.
- 예외 처리 시 PR 코멘트에 사유를 남깁니다.
