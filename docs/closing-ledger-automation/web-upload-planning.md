# 브랜아크 일일마감 자동화 웹 업로드 기획서

## 1. 최종 목표

아주 간단한 HTML 웹사이트에서 발주서 파일을 첨부하면, 파일을 Google Drive에 저장하고 발주서 데이터를 검증한 뒤 일일 마감 양식 Google Sheet에 내용을 추가한다.

```text
웹사이트 접속
→ 파일첨부
→ 파일 검증
→ Google Drive 저장
→ 발주서 내용 확인
→ 품목별 수량 집계
→ 단가표 매칭
→ 일일 마감 양식에 추가
→ 화면에서 검증 결과 확인
```

## 2. 운영 방향

사용자는 웹사이트에서 파일만 업로드한다. 실제 처리는 서버/API/GitHub Action/Google Sheets API에서 담당한다.

초기 목표는 아래 방식이다.

```text
파일첨부
→ Google Drive 발주서 폴더 저장
→ 해당 날짜 발주서 분석
→ 일일 마감 양식 파일에 append
```

기존에 논의했던 GitHub PR 검증 방식은 운영 안정화를 위해 유지한다. 다만 웹사이트에서는 사용자가 이해하기 쉽게 검증 결과를 바로 확인할 수 있게 한다.

## 3. 대상 파일 구조

### 발주서

- Google Drive 내 지정 폴더에 저장
- 예시 폴더: `TESTDRIVE/발주서`
- 파일명 예시: `브랜아크_맛집밥상_발주서_260702_발주용.xlsx`

### 일일 마감 양식

- Google Sheet 1개에 누적 반영
- 월별 시트 또는 일별 누적 시트 사용
- 예시 시트명: `일별정산_7월`

## 4. 업로드 검증 규칙

| 검증 항목 | 조건 | 처리 문구 |
|---|---|---|
| 파일명 중복 | Drive 폴더에 같은 파일명 존재 | 중복된 파일명입니다. |
| 파일 형식 오류 | xlsx, xls, csv 외 파일 | 지원하지 않는 파일 형식입니다. |
| 파일명 날짜 없음 | 파일명에서 YYMMDD 또는 YYYYMMDD 추출 불가 | 파일명에 날짜가 없습니다. |
| 운송장 시트 없음 | 발주서 내 `운송장` 시트 없음 | 운송장 시트를 찾을 수 없습니다. |
| 출고일지 시트 없음 | `출고일지` 또는 `출고일지(2)` 없음 | 출고일지 시트를 찾을 수 없습니다. |
| 상품명 컬럼 없음 | 운송장 G열 또는 `상품명1` 없음 | 상품명1 컬럼을 찾을 수 없습니다. |
| 출고일 없음 | 출고일지에서 출고일 확인 불가 | 출고일을 확인할 수 없습니다. |
| 단가 누락 | 단가표에 상품 없음 | 단가표에 없는 상품입니다. |
| 양식 변경 | 일일 마감 양식 필수 컬럼 누락 | 일일 마감 양식이 변경되었습니다. |

## 5. 발주서 처리 규칙

1. 운송장 시트의 `상품명1` 기준으로 품목을 집계한다.
2. 상품명 뒤 `_숫자`는 실제 수량으로 계산한다.
   - `아삭아삭 궁채장아찌 1kg_10` → `아삭아삭 궁채장아찌 1kg`, 수량 10
   - `김밥집 궁채 2kg_4` → `김밥집 궁채 2kg`, 수량 4
3. `_숫자`가 없으면 기본 수량은 1로 본다.
4. 출고일지와 운송장 집계 수량을 비교한다.
5. `출고일지`와 `출고일지(2)`가 함께 있으면 두 시트를 합산한다.
6. 주문일/출고요청일과 다르더라도 최종 마감 기준은 `출고일`이다.
7. 판매처 값이 있으면 판매처별 집계를 함께 만든다.
8. 판매처 값이 없으면 판매처별 집계는 `불가`로 표시하고 품목 마감만 진행한다.

## 6. 단가표 검증 규칙

단가표는 화면에 표 형식으로 전체 표시한다.

예시:

| 상품명 | 공급단가 |
|---|---:|
| 아삭아삭 궁채장아찌 4kg | 8,800 |
| 아삭아삭 궁채장아찌 1kg | 2,200 |
| 아삭아삭 궁채장아찌 10kg | 21,500 |
| 김밥집 궁채 2kg | 7,425 |

검증 문구는 아래처럼 보여준다.

```text
"아삭아삭 궁채장아찌 4kg" = 8,800원
"김밥집 궁채 2kg" = 7,425원
```

## 7. 일일 마감 양식 필수 컬럼

일일 마감 양식은 아래 컬럼을 기준으로 한다.

| 컬럼 | 설명 |
|---|---|
| 주문일 | 주문일 또는 참고 주문일 |
| 상품명 | 정규화된 상품명 |
| 수량 | 옵션 수량 반영 후 실제 수량 |
| 공급단가 | 단가표 기준 단가 |
| 공급가 합계 | 수량 × 공급단가 |
| 비고 | 검증 결과, 예외사항 |

## 8. 필수 운영 규칙

```text
* 일일 마감 양식을 마음대로 변경하지 마세요.
* 발주서 파일명에는 날짜를 포함해야 합니다.
* 운송장 시트의 상품명1 기준으로 수량을 계산합니다.
* 상품명 뒤 _숫자는 실제 수량으로 계산합니다.
* 마감 기준일은 출고일입니다.
* 단가표에 없는 상품은 자동 확정하지 않습니다.
* 검증 실패 시 일일 마감 양식에 반영하지 않습니다.
* 판매처 정보가 없으면 판매처별 집계는 생성하지 않습니다.
```

## 9. 자동화 구축 단계

### 1단계. 정적 HTML 화면 생성

- 파일 선택 UI
- 검증 결과 UI
- 단가표 UI
- 일일 마감 미리보기 UI

### 2단계. 백엔드 API 연결

- 파일 업로드 API
- Google Drive 저장 API
- Google Sheets 업데이트 API

### 3단계. GitHub Action 또는 서버 처리

- Drive 저장 완료 후 발주서 분석
- 검증 리포트 생성
- 오류 없을 때만 일일 마감 양식 append

### 4단계. 운영 안정화

- 중복 파일명 차단
- 처리 로그 저장
- 실패 내역 재처리
- 텔레그램 알림 연동

## 10. 사용자에게 보여줄 최종 결과

- 업로드 파일명
- 저장 위치
- 출고일
- 품목별 수량
- 판매처별 수량 가능 여부
- 단가표 매칭 결과
- 공급가 합계
- 일일 마감 양식 반영 여부
