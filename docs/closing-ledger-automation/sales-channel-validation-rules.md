# 판매처별 집계 및 검증 규칙

마감원장 자동화는 품목별 수량뿐 아니라 판매처별 판매 수량도 함께 확인해야 합니다.

---

## 1. 판매처 기준 컬럼 확인

발주서 또는 운송장 시트에서 A열 헤더가 아래 중 하나인지 확인합니다.

- `주문번호`
- `판매처`

### A열 헤더가 `판매처`인 경우

- A열 값을 판매처명으로 그대로 사용합니다.
- 예: 스마트스토어, 쿠팡, 식봄, 현대홈쇼핑, KB쇼핑 등

### A열 헤더가 `주문번호`인 경우

- 주문번호만으로 판매처를 확정할 수 없으면 자동 확정하지 않습니다.
- 같은 시트 안에 `판매처`, `채널`, `쇼핑몰`, `판매채널`, `주문경로` 컬럼이 있는지 추가 탐색합니다.
- 판매처 컬럼을 찾지 못하면 `판매처 확인 필요`로 표시하고 PR 검증 리포트에 노출합니다.

---

## 2. 판매처별 집계 기준

판매처가 확인되면 아래 기준으로 집계합니다.

1. 판매처별 전체 주문 건수
2. 판매처별 상품별 실제 수량
3. 판매처별 총 상품 수량
4. 판매처별 공급가 합계

옵션 수량은 기존 품목 집계와 동일하게 반영합니다.

예시:

| 판매처 | 원본상품명 | 정규화상품명 | 실제수량 |
|---|---|---|---:|
| 스마트스토어 | 아삭아삭 궁채장아찌 1kg_10 | 아삭아삭 궁채장아찌 1kg | 10 |
| 쿠팡 | 김밥집 궁채 2kg_4 | 김밥집 궁채 2kg | 4 |

---

## 3. 판매처별 요약 출력

검증 리포트와 최종 마감원장에는 아래 요약이 포함되어야 합니다.

| 판매처 | 주문건수 | 상품종류수 | 총수량 | 공급가 합계 | 확인필요 |
|---|---:|---:|---:|---:|---:|
| 스마트스토어 | 0 | 0 | 0 | 0 | 0 |
| 쿠팡 | 0 | 0 | 0 | 0 | 0 |
| 식봄 | 0 | 0 | 0 | 0 | 0 |

---

## 4. 품목별 + 판매처별 교차 검증

자동화는 아래 두 값이 반드시 일치하는지 확인합니다.

```text
전체 품목별 합산 수량 = 판매처별 품목 수량 합산
```

예시:

| 상품명 | 전체수량 | 판매처별 합산 | 검증상태 |
|---|---:|---:|---|
| 아삭아삭 궁채장아찌 1kg | 50 | 50 | 일치 |
| 김밥집 궁채 2kg | 6 | 5 | 불일치 |

불일치 발생 시 Google Spreadsheet 생성은 차단하고 PR에서 확인 필요로 표시합니다.

---

## 5. 판매처명 정규화 규칙

판매처명은 아래와 같이 정규화합니다.

| 입력값 예시 | 정규화 판매처명 |
|---|---|
| 스마트스토어 | 스마트스토어 |
| 네이버 | 스마트스토어 |
| N스토어 | 스마트스토어 |
| 쿠팡 | 쿠팡 |
| coupang | 쿠팡 |
| 식봄 | 식봄 |
| 현대 | 현대홈쇼핑 |
| 현대홈쇼핑 | 현대홈쇼핑 |
| KB | KB쇼핑 |
| KB쇼핑 | KB쇼핑 |

정규화되지 않은 판매처는 `판매처 확인 필요`로 표시합니다.

---

## 6. 오류 방지 강화 규칙

아래 조건은 자동화 오류 또는 검증 실패로 처리합니다.

| 조건 | 처리 |
|---|---|
| A열 헤더가 `주문번호` 또는 `판매처`가 아님 | 확인 필요 |
| A열이 `주문번호`인데 판매처 컬럼을 찾지 못함 | 확인 필요 |
| 판매처명이 비어 있음 | 오류 |
| 판매처명이 정규화 사전에 없음 | 확인 필요 |
| 판매처별 합산 수량과 전체 품목 수량이 다름 | 오류 |
| 판매처별 공급가 합계와 전체 공급가 합계가 다름 | 오류 |
| 출고일 기준 판매처별 집계가 불가능함 | 오류 |

오류 상태가 1건이라도 있으면 최종 Google Spreadsheet 생성은 진행하지 않습니다.

---

## 7. PR 검증 리포트 필수 항목 추가

PR 본문 또는 검증 리포트에는 아래 항목을 추가합니다.

- A열 헤더 확인 결과
- 판매처 컬럼 탐색 결과
- 판매처명 정규화 결과
- 판매처별 주문건수
- 판매처별 상품 수량
- 판매처별 공급가 합계
- 전체 품목 수량과 판매처별 합산 수량 일치 여부
- 전체 공급가 합계와 판매처별 공급가 합계 일치 여부

---

## 8. merge 전 체크리스트 추가

- [ ] A열 헤더가 `주문번호` 또는 `판매처`인지 확인

- [ ] 스마트스토어/쿠팡/식봄 등 판매처명이 정상 정규화되었는지 확인
- [ ] 판매처별 주문건수 확인
- [ ] 판매처별 품목 수량 확인
- [ ] 전체 품목 수량과 판매처별 합산 수량 일치 확인
- [ ] 판매처별 공급가 합계 확인
- [ ] 오류 또는 확인 필요 판매처가 없는지 확인
