BRANARK CLOSING LEDGER

브랜아크 일일마감 자동화

발주서 파일을 업로드하면 Google Drive에 저장하고, 단가표와 발주서를 검증한 뒤 일일 마감 양식에 추가하는 운영 기준 화면입니다.

필수 운영 규칙

일일 마감 양식을 마음대로 변경하지 마세요. 컬럼이 변경되면 자동화가 중단됩니다.

저장 및 반영 기준

발주서 업로드 파일은 Google Drive의 7월 / 발주서 폴더에 저장합니다.

업로드 전 같은 폴더에 동일 파일명이 있는지 확인하고, 중복이면 저장과 일일 마감 반영을 중단합니다.

검증 완료 데이터는 일일 마감 양식 시트에 주문일, 상품명, 수량, 공급단가, 공급가 합계, 비고 컬럼 기준으로 추가합니다.

공급단가는 공급단가.XLS 파일을 기준으로 매칭하며, 단가가 없는 상품은 자동 반영하지 않습니다.

발주서 분석 기준 시트는 운송장, 출고일지, 출고일지(2)입니다.

API env 연결 상태 기준

GOOGLE_SERVICE_ACCOUNT_JSON : Google Drive / Sheet API 인증 정보

DRIVE_FOLDER_ID : 발주서 저장 Google Drive 폴더

DAILY_SHEET_ID : 일일 마감 양식 Google Sheet

PRICE_SHEET_ID : 단가표 파일 또는 단가표 Sheet

연결 상태는 GitHub Actions의 Branark Closing Ledger API Env Check workflow에서 수동 점검합니다.

발주서 업로드

xlsx, xls, csv 파일만 지원합니다.

현재 화면은 업로드 API 연결을 전제로 한 HTML입니다. 실제 Drive 저장은 API 구현 후 동작합니다.

2

처리 상태

파일 업로드 후 단계별 상태를 표시합니다.

  1. 파일 형식 확인대기
  2. 중복 파일명 확인대기
  3. Google Drive 저장대기
  4. 발주서 내용 확인대기
  5. 단가표 매칭대기
  6. 일일 마감 양식 반영대기
