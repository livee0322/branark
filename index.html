브랜아크 발주서 업로드 및 일일마감 자동화

브랜아크 운영팀이 발주서 파일을 업로드하고, 실제 자동화 연결 상태와 테스트 반영 결과를 한 화면에서 확인하는 전용 페이지입니다.

현재 기능 범위

지금 단계에서는 발주서 파일을 Google Drive에 저장하고, 화면 입력값으로 일일 마감 양식 첫 번째 시트에 테스트 행 1건을 추가합니다. 실제 엑셀 발주서 분석은 아직 포함되지 않습니다.

0

자동화 연결 점검

페이지 로드 시 Apps Script health check를 실행해 Drive, 일일마감 시트, 단가표 접근 상태를 실제 응답 기준으로 표시합니다.

Apps Script 연결 상태 점검 중

브랜아크 Apps Script 응답과 Google 리소스 접근 상태를 확인하고 있습니다.

Google Drive 확인 전

발주서 저장 폴더 접근 상태를 점검합니다.

일일마감 시트 확인 전

테스트 행을 추가할 첫 번째 시트를 점검합니다.

단가표 확인 전

단가표 파일 또는 시트 접근 상태를 점검합니다.

이 화면은 URL 문자열만 보고 연결 성공을 표시하지 않습니다. Apps Script가 실제로 응답하고, 필요한 Drive/Sheet/단가표 설정이 모두 접근 가능할 때만 정상으로 표시합니다.

처리 흐름 정리

최종 자동화 이전 단계이므로, 현재 페이지가 실제로 무엇을 하고 무엇은 아직 하지 않는지 명확히 구분해 보여줍니다.

  1. 1. 발주서 파일 선택xlsx / xls / csv, 8MB 이하
  2. 2. 화면 입력값 검증주문일, 상품명, 수량, 공급단가 확인
  3. 3. Apps Script로 파일 전송hidden iframe / form POST 유지
  4. 4. Google Drive에 발주서 저장중복 파일명 정책 반영
  5. 5. 단가표 확인은 참고용현재는 접근/미리보기만 제공
  6. 6. 일일마감 시트에는 테스트 행만 추가실제 발주서 분석 반영은 아직 미구현
2

발주서 업로드 및 테스트 반영

실제 직원이 검토하기 쉽게 입력값과 파일 조건을 먼저 확인한 뒤 업로드를 시작합니다.

업로드 전 확인

중복 허용을 체크하지 않으면 같은 파일명은 Drive 저장 전에 차단됩니다.

화면의 상품명, 수량, 공급단가는 현재 테스트 행 작성용 입력값이며, 실제 발주서 내부 데이터를 읽어 계산하는 단계는 아직 아닙니다.

먼저 health check가 정상인지 확인한 뒤 업로드를 진행하세요.

3

  1. 파일 선택 및 형식 확인대기

  4. 중복 파일명 확인대기
  5. Google Drive 저장대기
  6. 일일마감 테스트 행 추가대기
